020package ch.powerunit.extensions.matchers;
021
022import java.lang.annotation.Documented;
023import java.lang.annotation.ElementType;
024import java.lang.annotation.Inherited;
025import java.lang.annotation.Retention;
026import java.lang.annotation.RetentionPolicy;
027import java.lang.annotation.Target;
028
029/**
030 * This annotation can be used on a java class, to mark this class as supporting
031 * generation of hamcrest matcher.
032 * <p>
033 * <b>This annotation is not supported on interface and enum. A error will be
034 * generated in this case.</b>
035 * <p>
036 * This annotation is processed by an annotation processor, in order to generate
037 * :
038 * <ul>
039 * <li>One class for each annotated classes, that will contains Hamcrest
040 * Matchers for the class.</li>
041 * <li>In case the annotation processor parameter
042 * "{@code ch.powerunit.extensions.matchers.provideprocessor.ProvidesMatchersAnnotationsProcessor.factory}"
043 * is set, this value define the fully qualified name of a interface that will
044 * be generated and will contains all <i>start method</i> allowing to create
045 * instance of the various matchers.</li>
046 * </ul>
047 * <p>
048 * <i>The generated classes are related with the hamcrest framework ; This
049 * library will be required in the classpath in order to compile or run the
050 * generated classes.</i>
051 * <p>
052 * <b>Concept regarding the generated Matchers</b>
053 * <p>
054 * Hamcrest Matchers can be used, for example, with test framework (JUnit,
055 * PowerUnit, etc.) to validate expectation on object. Hamcrest provides several
056 * matchers to validate some information of an object (is an instance of, is, an
057 * array contains some value, etc.), but can't provide ready to use matcher for
058 * your own object. When trying to validate properties of object, no syntaxic
059 * sugar (ie. autocompletion) are available and only the generic method can be
060 * used.
061 * <p>
062 * With this annotation, it is possible to provide <i>builder-like</i> method,
063 * based on hamcrest, to validate fields of an object. To do so, the annotation
064 * processor do the following :
065 * <ul>
066 * <li>For each public field or public method starting with {@code get} or
067 * {@code is} without any argument, generated a private matcher based on the
068 * {@code org.hamcrest.FeatureMatcher} for this field ; this will provide a way
069 * to validate the value of one specific <i>property</i>.</li>
070 * <li>Generate an interface and the related implementation of a matcher (which
071 * is also a builder) on the annotated classes itself, which will validate all
072 * of the <i>properties</i>.</li>
073 * <li>Generate various methods, with a name based on the annotated class, to
074 * start the creation of the matcher.</li>
075 * </ul>
076 * The annotation processor will also generate javadoc and try to retrieve from
077 * the javadoc of the annotated element the information regarding generic
078 * attribute.
079 * <p>
080 * The processor will also try, for field which are not based on generics, to
081 * link the generated matchers between them, in order to provide chaining of the
082 * Fields.
083 * <p>
084 * <i>First example</i>
085 * <p>
086 * Let's assume the following class, containing one single field, will be
087 * processed by the annotation processor :
088 * 
089 * <pre>
090 * package ch.powerunit.extensions.matchers.samples;
091 *
092 * import ch.powerunit.extensions.matchers.ProvideMatchers;
093 *
094 * &#64;ProvideMatchers
095 * public class SimplePojo {
096 *      public String oneField;
097 * }
098 * </pre>
099 * 
100 * In this case a class named {@code SimplePojoMatchers} will be generated. As a
101 * public interface, the following methods will be available :
102 * <ul>
103 * <li>{@code public static SimplePojoMatcher simplePojoWith()}: This will
104 * return a matcher (see below), which by default matches any instance of the
105 * SimplePojo class.</li>
106 * <li>
107 * {@code public static SimplePojoMatcher simplePojoWithSameValue(SimplePojo  other)}
108 * : This will return a matcher, which by default matches an instance of the
109 * SimplePojo having the field {@code oneField} matching (Matcher {@code is} of
110 * hamcrest) of the reference object.</li>
111 * </ul>
112 * The returned interface is already a correct hamcrest matcher. This interface
113 * provide method that set the expectating on the various fields. As in this
114 * case, where is only one field, the returned interface ensure that once the
115 * expected is defined, it is not possible to modify it. Depending of the type
116 * of the field, various methods are generated to define the expectation :
117 * <ul>
118 * <li>Two standards methods are defined for all type of fields : {@code Matcher
119 * <SimplePojo> oneField(Matcher<? super java.lang.String> matcher)} and
120 * {@code Matcher<SimplePojo> oneField(String value)}. The second one is a
121 * shortcut to validate the field with the {@code is} Matcher and the first one
122 * accept another matcher ; The method with matcher parameter ensures that it is
123 * possible to combine any other matcher provided by hamcrest or any others
124 * extensions.
125 * <li>As the field is a String, others special expectation (shortcut) are
126 * provided, for example : {@code oneFieldComparesEqualTo},
127 * {@code oneFieldLessThan}, {@code oneFieldStartsWith}, etc.</li>
128 * </ul>
129 * <i>Second example</i>
130 * <p>
131 * In case the annotated contains several fields, the generated <i>DSL</i>
132 * provide chained methods, for example
133 * {@code TwoFieldsPojoMatcher firstField(Matcher<? super String> matcher)} and
134 * {@code TwoFieldsPojoMatcher secondField(Matcher<? super String> matcher)}.
135 * 
136 * Also, depending on the class, other <i>with</i> methods may be provided.
137 * <p>
138 * <i>Usage example</i>
139 * <p>
140 * Assuming powerunit as a test framework, the usage of the matcher will look
141 * like :
142 * 
143 * <pre>
144 * &#64;Test
145 * public void testOKMatcherWithComparable() {
146 *      Pojo1 p = new Pojo1();
147 *      p.msg2 = "12";
148 *      assertThat(p).is(Pojo1Matchers.pojo1With().msg2ComparesEqualTo("12"));
149 * }
150 * 
151 * </pre>
152 * 
153 * Assuming the {@code msg2} is change to the value {@code 11}, the resulting
154 * unit test error will look like (the Pojo1 classes contains several fields) :
155 * 
156 * <pre>
157 * expecting an instance of ch.powerunit.extensions.matchers.samples.Pojo1 with
158 * [msg2 a value equal to "12"]
159 * [msg3 ANYTHING]
160 * [msg4 ANYTHING]
161 * [msg5 ANYTHING]
162 * [msg6 ANYTHING]
163 * [msg7 ANYTHING]
164 * [msg8 ANYTHING]
165 * [msg9 ANYTHING]
166 * [msg12 ANYTHING]
167 * [msg1 ANYTHING]
168 * [myBoolean ANYTHING]
169 * [oneBoolean ANYTHING]
170 * but [msg2 "11" was less than "12"]
171 * 
172 * </pre>
173 * 
174 * <hr>
175 * <p>
176 * <b>Overriding the way the matchers are generated</b>
177 * <ul>
178 * <li>The attribute {@link #matchersClassName() matchersClassName} may be used
179 * to change the simple name (<b>NOT THE FULLY QUALIFIED NAME</b>) of the
180 * generated class.</li>
181 * <li>The attribute {@link #matchersPackageName() matchersPackageName} may be
182 * used to change the package name of the generated class.</li>
183 * </ul>
184 * <hr>
185 * <p>
186 * <b>Extensions</b>
187 * <p>
188 * The framework, since version 0.1.0, is able to detect others library and use
189 * it :
190 * <ul>
191 * <li>If <a href="https://github.com/exparity/hamcrest-date">Hamcrest Date</a>
192 * is available, additional DSL method are added for the Java 8 Date objects.
193 * </li>
194 * <li>If <a href="https://github.com/NitorCreations/matchers">Hamcrest 1.3
195 * Utility Matchers</a> is available, additional DSL method are added for the
196 * collections.</li>
197 * <li>If <a href="https://github.com/spotify/java-hamcrest">Spotify Hamcrest
198 * (jackson)</a> is available and the {@link #JSON_EXTENSION} is used on
199 * {@link #extensions()}, then method to validate String as json are added.</li>
200 * <li>If <a href="https://github.com/orien/bean-matchers">Bean Matchers</a> is
201 * available additional method to validate Class field.</li>
202 * </ul>
203 * <hr>
204 * <p>
205 * <b>Linking between matchers</b>
206 * <p>
207 * The annotation processor try to link the generated matchers. The goal is that
208 * when a field or the super class also have a matchers, this one is used. It is
209 * not always possible to link everything but the concept is the following :
210 * <ul>
211 * <li>If the super class is compiled at the same time and is annotated with
212 * this annotation, the annotation processor may add a control on the parent and
213 * generated chaining method.</li>
214 * <li>If a field class is compiled at the same time and is annotated with this
215 * annotation, the annotation processor may add chaining and dedicated control
216 * that use this matcher.</li>
217 * <li>If a matcher, following the convention of this framework is detected for
218 * the super class or a field (already compiled for example), the annotation
219 * processor may also apply the previous rules by using this detected
220 * matcher.</li>
221 * <li>The annotation processor may also try to find matcher for element of Map,
222 * List, Array.</li>
223 * </ul>
224 * <hr>
225 * <p>
226 * <b>Ignore field in <i>WithSameValue</i> matchers</b>
227 * <p>
228 * Since version 0.3.0, it is possible to pass a list of field names to be
229 * ignored when constructing a <i>WithSameValue</i> matcher. The generated
230 * matcher tries to also apply in cascade this ignore feature to included
231 * object. The syntax to ignore fields of another fields is to use
232 * <code>fieldName.fieldNameInside</code>.
233 * <p>
234 * The generated matcher may not be able to apply the ignore feature in all the
235 * included fields of the fields.
236 * <hr>
237 * <p>
238 * <b>Cycle detection</b>
239 * <p>
240 * Since version 0.3.0, the generated <i>WithSameValue</i> matchers try to
241 * detect the cycle in the object to be compared. Cycle may be included for
242 * example when working with bidirectional link. The matcher is <b>not</b> able
243 * to detect all cycles, but when detected, replace the control by a same
244 * instance control.
245 * 
246 * @author borettim
247 *
248 */
249@Documented
250@Retention(RetentionPolicy.SOURCE)
251@Target({ ElementType.TYPE })
252@Inherited
253public @interface ProvideMatchers {
254        /**
255         * This attribute may be used to override the default class name that will
256         * contains the generated matchers.
257         * <p>
258         * <i>By default, this attribute is an empty string, which indicate to use the
259         * default construction pattern.</i>
260         * <p>
261         * By default, the Matchers class name is the name of the annotated class,
262         * followed by {@code Matchers}.
263         * <p>
264         * <i>Using this attribute may make the matcher undetectable when working with
265         * separated compilation.</i>
266         * 
267         * @return the name of the matchers class or an empty string if this is not
268         *         overloaded.
269         */
270        String matchersClassName() default "";
271
272        /**
273         * This attribute may be used to override the default package name that will be
274         * used for the generated matchers.
275         * <p>
276         * <i>By default, this attribute is an empty string, which indicate to use the
277         * default construction pattern.</i>
278         * <p>
279         * By default, the Matchers package name is the same that the annotated class.
280         * <p>
281         * <i>Using this attribute may make the matcher undetectable when working with
282         * separated compilation.</i>
283         * 
284         * @return the name of the matchers package or an empty string if this is not
285         *         overloaded.
286         */
287        String matchersPackageName() default "";
288
289        /**
290         * This attribute may be used to set a comments that will be passed inside the
291         * {@link javax.annotation.Generated#comments() comments} attribute of the
292         * {@link javax.annotation.Generated @Generated} annotation.
293         * 
294         * @return the comments or an empty string if ignored.
295         */
296        String comments() default "";
297
298        /**
299         * This attribute may be used to generate additional exposition methods for the
300         * object.
301         * <p>
302         * By default, only the standard method are generated.
303         * 
304         * @return additional method to be generated or an empty array by default.
305         * @since 0.1.0
306         */
307        ComplementaryExpositionMethod[] moreMethod() default {};
308
309        /**
310         * This attribute may be used to enable some extensions for this objects.
311         * <p>
312         * By default, no extension are enabled.
313         * 
314         * @return the extension to be used or an empty array by default.
315         * @since 0.1.0
316         * @see #JSON_EXTENSION An extension to add a validation on String, as JSON,
317         *      based on the <a href="https://github.com/spotify/java-hamcrest">Spotify
318         *      Matchers</a>.
319         */
320        String[] extensions() default {};
321
322        /**
323         * This attribute may be used in case when it is not possible to create a
324         * <i>sure</i> <code>WithSameValue</code> matcher. This may be the case when the
325         * annotated class extends a not annotated class or when the parent class is
326         * already compiled.
327         * <p>
328         * By default, this mode is set to false.
329         * <p>
330         * In case this option is set to true and the <code>WithSameValue</code> matcher
331         * is considered as weak, a warning will be produced.
332         * 
333         * @return to create (or not) a <code>WithSameValue</code> matcher even if it is
334         *         not possible to ensure the control of the field of the parent class.
335         * @since 2.0.0
336         */
337        boolean allowWeakWithSameValue() default false;
338
339        /**
340         * May be use as part of {@link #extensions()} to enable, if available, an
341         * extension an each String to be validated by using a json validation.
342         * 
343         * 
344         * @since 0.1.0
345         */
346        public static final String JSON_EXTENSION = "json-extension";
347}
